# 第 7 章：多路召回管线

## 7.1 为什么需要多路召回

单一检索方法在面对多样化的用户查询时往往表现不稳定。向量检索擅长语义匹配但可能在精确关键词匹配上失准，关键词检索（BM25）在精确匹配上表现出色但无法理解同义词和上下文，知识图谱检索能提供结构化推理但覆盖面有限。多路召回（Multi-Recall）的核心思想是：同时运行多种互补的检索策略，然后将结果融合，取各策略之长。

## 7.2 检索策略详解

### 7.2.1 稠密向量检索

稠密向量检索（Dense Retrieval）将查询和文档都映射到同一个向量空间，通过计算余弦相似度来检索最相关的文档。这是 RAG 系统中最核心的检索方式。

实现稠密向量检索的关键组件：
1. **嵌入模型**：将文本转换为向量
2. **向量索引**：加速最近邻搜索（如 HNSW、IVF）
3. **相似度计算**：通常使用余弦相似度或内积

稠密检索的优势在于语义理解能力——能匹配"如何配置 VPN"和"VPN 设置步骤"这样的同义表达。缺点是对罕见词和专有名词的匹配不够精确。

### 7.2.2 稀疏检索（BM25）

BM25 是基于词频-逆文档频率（TF-IDF）改进的检索算法，它计算查询词在文档中出现的频率和重要性。BM25 的核心公式：

- TF 项：考虑词频的饱和效应（某个词出现太多次后的边际收益递减）
- IDF 项：衡量词的区分能力（常见词的权重低，罕见词的权重高）
- 文档长度归一化：避免长文档天然获得更高分数

BM25 的优势在于精确匹配和可解释性——关键词匹配一目了然，结果容易调试。缺点是无法处理同义词和语义相近但措辞不同的查询。

### 7.2.3 知识图谱检索

当用户问"恒瑞医药生产哪些药品？"时，这不是一个文本匹配问题，而是一个图结构查询问题。知识图谱检索通过实体链接和关系遍历来回答这类问题：

1. 识别查询中的实体（"恒瑞医药"）
2. 在图谱中找到该实体
3. 沿「生产」关系遍历到药品实体
4. 返回关联的药品列表

知识图谱检索的优势在于精确性和可解释性——答案有明确的关系路径可追溯。缺点是受限于图谱的覆盖范围，对未知实体无能为力。

### 7.2.4 结构化检索

结构检索基于文档的元数据字段（部门、类型、日期等）进行过滤和排序。例如："只检索采购部门的文档"或"只检索 2024 年之后的文档"。结构化检索通常不作为独立检索通道，而是与其他检索方法结合使用——先用结构化条件缩小候选集，再进行语义匹配。

## 7.3 结果融合策略

多路检索产生多组结果后，如何将它们融合为一个统一的排名列表？

### 7.3.1 加权融合

最简单的融合方式是为每个检索通道分配一个权重，然后将归一化后的分数加权求和：

```
final_score = w_vector × score_vector + w_bm25 × score_bm25 + w_kg × score_kg
```

权重的确定可以通过网格搜索或贝叶斯优化。实践中，稠密检索的权重通常最高（0.3-0.5），关键词检索次之（0.2-0.3），知识图谱和结构化检索较低。

### 7.3.2 RRF 融合

倒数排名融合（Reciprocal Rank Fusion, RRF）不依赖分数绝对值，而是基于排序位置：

```
RRF_score(d) = Σ 1 / (k + rank_i(d))
```

其中 k 是一个常数（通常为 60），rank_i(d) 是文档 d 在第 i 个检索结果列表中的排名位置。RRF 的优势是无需对分数进行归一化，对不同检索方法的分数尺度不敏感。

### 7.3.3 学习式融合

在有标注数据的场景下，可以使用学习排序（Learning to Rank, LTR）方法来学习最优的融合权重。常用的模型包括 LambdaRank、LightGBM 等。LTR 的效果通常优于固定权重和 RRF，但需要足够的标注数据。

## 7.4 重排序（Reranking）

多路召回的目标是"不遗漏"，重排序的目标是"排准确"。召回阶段通常返回 top-50 甚至 top-100 的结果，而重排序阶段使用更精确（也更慢）的 Cross-Encoder 模型对这组结果重新打分：

1. **召回阶段**：使用轻量级的双编码器（Bi-Encoder）快速检索候选集
2. **重排阶段**：使用精确的交叉编码器（Cross-Encoder）对候选集重新排序

Cross-Encoder 将查询和文档拼接后一起输入模型，计算相关性分数。由于需要逐对计算，延迟比双编码器高一个数量级，但精度也显著更高。因此重排阶段的候选集大小需要严格控制（通常 ≤50）。

## 7.5 本章小结

多路召回管线是 RAG 系统的检索核心，通过组合互补的检索策略和精细的结果融合机制，在召回率和准确率之间取得最佳平衡。本章介绍的四种检索策略（稠密检索、稀疏检索、知识图谱检索、结构化检索）和三种融合方法（加权融合、RRF、LTR）构成了多路召回的基础框架。

完整的可运行代码示例见 `demos/ch07/multi_recall_pipeline.py`，它演示了从多路检索到分数融合再到重排序的完整流程。
