# 第 11 章：层级检索与混合检索

## 11.1 单一检索方法的局限性

前几章介绍了向量检索、关键词检索和知识图谱检索各自的工作原理。但在实际应用中，单一检索方法往往难以应对多样化的查询需求。稠密向量检索在语义匹配上表现出色，但在处理精确关键词匹配（如产品型号、专有名词）时力不从心；BM25 精确匹配能力强，但无法理解同义词和上下文；知识图谱检索能处理结构化查询，但覆盖范围受限于图谱的构建质量。

层级检索与混合检索正是为了解决这个问题而设计的。它们通过组合多种检索策略，取长补短，在不同维度上覆盖用户查询的语义空间。

## 11.2 稠密检索与稀疏检索

### 11.2.1 稠密向量检索

稠密检索（Dense Retrieval）使用神经网络模型将文本映射到连续的向量空间中。在这种表示下，语义相似的文本在向量空间中距离更近。它的核心优势是语义理解能力——能匹配"笔记本"和"笔记本电脑"这样的同义表达。

稠密检索的弱点是对罕见词和拼写变体不敏感。例如，如果知识库中包含产品型号"ZETA-X2000"，而用户搜索"zeta x2000"或"ZETA X-2000"，向量检索可能无法精确匹配。

### 11.2.2 稀疏检索（BM25）

稀疏检索（Sparse Retrieval）基于词袋模型（Bag of Words），通过计算查询词与文档的词频统计相关性来进行检索。BM25 是最经典的稀疏检索算法，它考虑了词频饱和效应和文档长度归一化。

稀疏检索的优势是精确性和可解释性——匹配结果就是包含查询关键词的文档。弱点是对同义词和语义近似无能为力。

## 11.3 混合检索策略

### 11.3.1 加权融合

最直接的混合方式是加权融合（Weighted Fusion），将稠密检索和稀疏检索的分数通过权重组合：

```python
final_score = alpha * dense_score + (1 - alpha) * sparse_score
```

α 控制两种检索方法的贡献比例。α=1 时退化为纯稠密检索，α=0 时退化为纯 BM25。最佳 α 值可以通过在验证集上搜索确定，通常在 0.3-0.7 之间。

### 11.3.2 RRF 融合

倒数排名融合（Reciprocal Rank Fusion, RRF）不依赖原始分数，而是基于排序位置进行融合。对于每个文档，RRF 计算其在所有检索结果列表中的排序位置的倒数：

```python
rrf_score(d) = Σ 1 / (k + rank_i(d))
```

RRF 的优势是对不同检索方法的分数尺度不敏感——不需要对分数进行归一化。常数 k 控制排序位置的衰减速度，k 越小高位排序的影响越大。推荐值 k=60。

### 11.3.3 自适应混合

自适应混合（Adaptive Hybrid）根据查询的特性动态调整融合策略。例如，包含产品型号的查询应该增加稀疏检索的权重，而语义模糊的问题则应该增加稠密检索的权重。

```python
def adaptive_weight(query):
    # 如果查询包含数字、大写字母组合（疑似产品型号），增加稀疏权重
    if re.search(r'[A-Z]+-\d+', query):
        return 0.3  # 稠密权重降低
    # 如果查询较长（含描述性内容），增加稠密权重
    if len(query) > 20:
        return 0.7
    return 0.5  # 默认平衡
```

## 11.4 Small-to-Big 检索

Small-to-Big 检索是一种解决"块大小矛盾"的策略。检索时使用小块（Small Chunk）以获得更精准的匹配，而生成答案时提供包含小块上下文的更大文本块（Big Chunk）：

1. 将文档按小粒度切分（如 120 字符的小块）
2. 对小块构建向量索引
3. 检索时在小块中搜索，找到最相关的候选
4. 将每个候选小块映射回其所属的大块（如 600 字符的段落）
5. 将大块作为上下文送入 LLM

这种策略的优点是：小块的检索精度更高（减少噪声），而大块的上下文更完整（有利于 LLM 理解）。检索精度和上下文完整性不再是一对矛盾。

## 11.5 Step-back Prompting

Step-back Prompting 是另一种增强检索质量的策略。它的核心思想是：当直接检索效果不佳时，先生成一个更抽象、更通用的"退一步"问题，用这个退一步问题的结果来增强原始问题的检索。

例如，原始问题是"恒瑞医药的卡瑞利珠单抗的适应症是什么？"，退一步问题可能是"PD-1 抑制剂类药物的常见适应症有哪些？"。退一步问题的检索结果提供了更广泛的知识背景，与原始问题的精确检索结果互补。

Step-back 的实现通常需要一个小型 LLM 来生成退一步问题，或者使用预定义的规则模板。

## 11.6 本章小结

层级检索与混合检索通过组合多种检索策略，解决了单一检索方法的局限性。本章介绍了稠密检索与稀疏检索的加权融合、RRF 融合、自适应混合、Small-to-Big 检索和 Step-back Prompting 等核心技术。在实际系统中，建议从简单的加权融合开始，根据具体的业务需求和数据特征逐步引入更复杂的策略。

完整的可运行代码示例见 `demos/ch11/hybrid_search.py`，它演示了稠密检索、稀疏检索、RRF 融合、Small-to-Big 检索和 Step-back Prompting 的完整实现。
