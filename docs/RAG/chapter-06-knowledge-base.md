# 第 6 章：知识库构建

## 6.1 知识库的分层架构

一个生产级的 RAG 知识库不是简单的"文档集合"，而是一个多层级、多维度的知识组织体系。从下到上可以分为四个层次：

**原始文档层**是最底层的存储，保存所有原始格式的文档。这一层不关心文档内容的结构，只负责完整性和可追溯性。当上层引用出现问题时，可以回溯到原始文档进行核实。

**文本块层**是经过解析和切分后的文本片段。每个文本块携带文档来源、章节路径、块序号等元数据。这一层是向量检索的直接对象。

**向量索引层**将文本块转换为嵌入向量并构建索引。这一层负责语义相似度的计算，是整个 RAG 系统的核心检索通道。

**结构化知识层**（可选）包含知识图谱、实体索引、关系索引等结构化表示。这一层提供精确查询和多跳推理能力，是对纯向量检索的补充和增强。

## 6.2 数据源接入策略

企业知识库的数据源往往是异构的、分布式的。数据接入策略需要解决以下问题：

### 6.2.1 全量导入 vs 增量更新

**全量导入**适用于系统初始化和周期性重建。全量导入的优势是可以重新优化索引结构，消除数据碎片。缺点是成本高、耗时长，在大规模场景下可能需要数小时甚至数天。

**增量更新**适用于日常的数据同步。增量更新只处理新增或变更的文档，效率高但可能引入索引碎片。长期只增量不重建会导致索引质量下降。

推荐策略是"每日增量、每周全量"：工作日使用增量更新保持数据新鲜度，周末使用全量重建优化索引质量。

### 6.2.2 数据源变更检测

对于持续接入的数据源，需要一种高效的变更检测机制。常用的策略包括：

- **时间戳比较**：记录每个文档的最后修改时间，定期检查是否有更新
- **文件哈希比对**：计算文件的 MD5 或 SHA256 哈希值，比对发现变更
- **事件监听**：通过文件系统监听（inotify/Watcher）或数据库变更数据捕获（CDC）实时感知变更

## 6.3 文档质量评估

不是所有文档都适合加入知识库。在文档被索引之前，需要进行质量评估和筛选：

- **完整性检查**：文档是否包含完整的信息，是否存在大段缺失
- **时效性评估**：文档的创建时间和最后更新时间，过时的文档应被标记或排除
- **权威性评级**：不同来源的文档有不同的可信度，官方文档优先级高于社区内容
- **内容密度分析**：有效信息与噪声的比例，低密度文档可能不值得索引

```python
def assess_document_quality(doc):
    scores = {}
    # 完整性：检查内容长度是否满足最低要求
    scores["completeness"] = min(len(doc.content) / 100, 1.0)
    # 时效性：越新的文档分数越高
    age_days = (datetime.now() - doc.created_at).days
    scores["freshness"] = max(0, 1 - age_days / 365)
    # 内容密度：有效内容占比
    total_chars = len(doc.content)
    meaningful_chars = len(re.sub(r"\s+", "", doc.content))
    scores["density"] = meaningful_chars / total_chars if total_chars > 0 else 0
    return scores
```

## 6.4 数据版本管理

知识库的版本管理是生产环境中经常被忽视但至关重要的环节。当知识更新导致答案变化时，需要有能力追溯历史版本、进行 A/B 测试或在必要时回滚。

实现版本管理的方式：
- **快照机制**：每次全量重建时保存完整快照，支持按时间点回滚
- **变更日志**：记录每次数据变更的详情（变更时间、变更类型、影响范围）
- **版本标签**：为每个索引版本分配标签（如 v1.0、v2.0），查询时可以指定使用哪个版本

版本管理与 A/B 测试结合使用：让一部分流量走旧版本，另一部分走新版本，对比两个版本的检索效果和用户满意度。

## 6.5 知识库的维护与监控

知识库不是"建好就不用管"的。持续的维护和监控是保证系统质量的关键：

- **索引健康检查**：定期检查向量索引的查询延迟和召回率是否在正常范围内
- **数据一致性校验**：随机抽样检查文本块内容和检索结果是否匹配
- **覆盖率分析**：统计分析知识库对常见问题的覆盖程度，识别知识盲区
- **数据清理**：定期清理过期、重复或低质量的文档，控制知识库规模

## 6.6 本章小结

知识库构建是 RAG 系统中投入最大也最需要持续维护的环节。一个高质量的知识库需要合理的分层架构、灵活的数据接入策略、严格的质量评估和持续的系统维护。本章介绍的知识库构建原则和方法，为后续的检索系统搭建奠定了数据基础。
