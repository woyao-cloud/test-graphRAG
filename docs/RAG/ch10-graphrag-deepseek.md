# 第 10 章：GraphRAG 深度实践

## 10.1 从传统 RAG 到 GraphRAG

传统 RAG 系统将文档切分为文本块后进行向量检索，这种方式虽然有效，但存在两个根本性局限：第一，它无法理解实体之间的多跳关系——"紫杉醇的供应链是怎样的？"这类问题需要跨多个文本块的推理；第二，它缺乏全局视角——"这个数据集的主要主题是什么？"这类总结性问题需要理解整个文档集的宏观结构。

GraphRAG（Graph-based Retrieval-Augmented Generation）通过引入知识图谱来解决这些问题。与传统的"向量化+检索"范式不同，GraphRAG 在索引阶段增加了实体抽取、关系抽取、社区检测和社区摘要等步骤，构建了一个结构化的知识图谱。检索时，系统可以在图谱上进行多跳推理，也可以从社区报告中获取全局视角。

## 10.2 GraphRAG 索引流水线

GraphRAG 的索引流水线是其核心创新，它将原始文本转化为结构化的图数据。完整的索引流程包含以下五个主要步骤：

### 10.2.1 文本分块

与标准 RAG 一样，GraphRAG 首先将文档切分成文本单元（Text Units）。但与标准 RAG 不同的是，文本单元不仅是检索的基本单位，也是实体抽取的基本上下文窗口。文本单元的大小直接影响实体抽取的质量——太大的块可能导致实体共现关系虚高，太小的块可能导致关系被遗漏。

### 10.2.2 实体与关系抽取

这是 GraphRAG 最关键的一步。系统对每个文本单元使用 LLM 进行实体和关系抽取，从非结构化文本中提取出结构化的（头实体、关系、尾实体）三元组。

抽取质量直接影响后续所有环节的效果。为了提升抽取质量，GraphRAG 引入了 Gleaning 机制——对同一个文本单元进行多轮抽取，逐步补充遗漏的实体和关系。每轮 Gleaning 都会检查前一轮的结果，识别可能遗漏的实体。

Gleaning 轮数的选择需要在质量和成本之间权衡：
- 0 轮（无 Gleaning）：成本最低，但可能遗漏 20-30% 的关系
- 1-2 轮：推荐的默认值，能捕获大部分实体关系
- 3+ 轮：边际收益递减，仅在精度敏感场景使用

### 10.2.3 图谱构建

将抽取到的实体和关系组合成一个完整的知识图谱。这个步骤包括：
- 实体去重和融合：不同文本单元中提到的同一实体需要合并
- 关系聚合：同一对实体之间的多条关系需要合并和加权
- 构建邻接表：支持高效的图遍历操作

### 10.2.4 社区检测

社区检测（Community Detection）是 GraphRAG 的核心创新之一。它将知识图谱划分为多个社区（Community），每个社区包含一组高度关联的实体。GraphRAG 使用莱顿算法（Leiden Algorithm）进行层次化社区检测，生成多个层级的社区结构：

- Level 0：细粒度社区，每个社区包含紧密关联的实体（如"恒瑞医药及其产品线"）
- Level 1：中等粒度社区，将相关的细粒度社区合并（如"医药企业集群"）
- Level 2：粗粒度社区，覆盖更大的主题范围（如"整个医药供应链"）

层次化社区结构的价值在于，不同粒度的社区适用于不同类型的问题。细粒度社区适合具体事实查询，粗粒度社区则适合全局总结性问题。

### 10.2.5 社区摘要

对每个检测到的社区，GraphRAG 使用 LLM 生成自然语言摘要。社区摘要包含该社区的主题、关键实体、核心关系链和重要发现。这些摘要在后续的全局搜索（Global Search）中扮演核心角色。

## 10.3 GraphRAG 查询引擎

### 10.3.1 Local Search（本地搜索）

Local Search 适用于具体的事实查询，如"恒瑞医药生产哪些药品？"。它的工作流程是：

1. 将用户查询语义匹配到相关的实体
2. 以匹配到的实体为起点，在图谱中进行 1-2 跳扩展
3. 收集扩展路径上的实体、关系和原始文本单元
4. 将收集到的上下文送入 LLM 生成答案

Local Search 的优势是精度高——答案直接基于图谱中的结构化关系。缺点是视角有限——只能看到局部子图。

### 10.3.2 Global Search（全局搜索）

Global Search 适用于总结性和趋势性问题，如"这个数据集主要讨论什么？"。它的工作流程采用了 Map-Reduce 架构：

1. **Map 阶段**：将用户查询与所有社区摘要进行语义匹配，找到最相关的社区
2. **Reduce 阶段**：将匹配到的社区摘要和关键发现汇总，生成综合答案

Global Search 的优势是有全局视角，能回答需要跨文档综合分析的问题。

### 10.3.3 DRIFT Search（层次化遍历搜索）

DRIFT（Dynamic Retrieval of Information via Focused Traversal）Search 是 GraphRAG 中最复杂的查询模式，专门用于多跳推理问题，如"紫杉醇的完整供应链是怎样的？"。

DRIFT Search 的核心思想是"从起点社区开始，在社区间动态跳转，逐步构建推理路径"。它的工作流程是：

1. 找到与问题最相关的起点社区
2. 在起点社区内执行广度优先遍历
3. 当遍历跨越社区边界时，跳转到相邻社区继续探索
4. 将遍历路径合并为完整的推理链

### 10.3.4 Basic Search（基础搜索）

Basic Search 是纯文本单元搜索，不涉及知识图谱。它作为兜底方案，当其他搜索模式都无法提供答案时使用。

## 10.4 查询模式选择指南

| 问题类型 | 推荐模式 | 示例 |
|---|---|---|
| 具体事实查询 | Local | "恒瑞医药生产哪些药品？" |
| 全局总结 | Global | "这个数据集的主要主题是什么？" |
| 多跳推理 | DRIFT | "紫杉醇的完整供应链是怎样的？" |
| 简单文本匹配 | Basic | "文档中提到了哪些监管机构？" |

在实际系统中，可以配置自动路由规则，根据问题中的关键词自动选择最合适的查询模式。

## 10.5 GraphRAG + DeepSeek 集成方案

将 GraphRAG 与 DeepSeek 集成可以充分发挥两者的优势：

- **DeepSeek 作为实体抽取引擎**：DeepSeek 的强项在于从非结构化文本中提取结构化信息，非常适合 GraphRAG 的实体和关系抽取阶段。
- **DeepSeek 作为社区摘要生成器**：社区摘要需要较强的总结概括能力，DeepSeek 在这类任务上表现优异。
- **DeepSeek 作为答案生成器**：在查询阶段，DeepSeek 可以根据 GraphRAG 提供的结构化上下文生成高质量的最终答案。

配置示例：在 GraphRAG 的 settings.yaml 中将 chat_model 设为 DeepSeek 模型，使用 DeepSeek 完成所有 LLM 调用。

## 10.6 本章小结

GraphRAG 通过在传统 RAG 之上叠加知识图谱层，显著增强了系统的推理和总结能力。本章深入讲解了 GraphRAG 的索引流水线（分块、实体抽取、图谱构建、社区检测、社区摘要）和四种查询模式（Local、Global、DRIFT、Basic），并介绍了与 DeepSeek 的集成方案。

完整的可运行代码示例见 `demos/ch10/graphrag_deepseek.py`，它演示了简化版 GraphRAG 流水线的完整实现，包含实体抽取、图谱构建、社区检测和四种查询模式。
